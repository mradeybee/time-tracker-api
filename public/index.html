    Running The Application

    • Clone or download the app from github

    $ git clone git@github.com:mradeybee/time-tracker-api.git

    • Navigate to the app directory

    $ cd time-tracker-api

    • Bundle dependencies

    $ bundle install

    • Run Database setup

    $ rails db:create && rails db:migrate

    • Setup environment variables

    $ export SECRET="A SECRETE TOKEN"

    • Start the application

    $ rails server

    Running Tests

    The tests are run using RSpec.

    $ rspec

    Signing Up

    The sign up endpoint takes a valid email and a password as the request payload, this creates a user record and returns an auth token.

    Request

    endpoint: POST '/users'

payload: { auth: { email: 'valid@email.com', password: 'securepassword' } }

    Response

    status: 201

body: { jwt: 'valid auth token', refresh_token: 'valid refresh token' }

    Login

    The login endpoint takes a valid email and a password as the request payload, this logs a user in and returns an auth token if the login cridentials are correct.

    Request With Valid Cridentials

    endpoint: POST '/auth/login'

payload: { auth: { email: 'valid@email.com', password: 'securepassword' } }

    Response

    status: 200

body: { jwt: 'valid auth token', refresh_token: 'valid refresh token' }

    Request With Wrong Cridentials

    endpoint: POST '/auth/login'

payload: { auth: { email: 'valid@email.com', password: 'wrongpassword' } }

    Response

    status: 422

body: { errors: 'Invalid Cridentials' }

    Logout

    The logout endpoint takes the logged in user's refresh token as the request payload, this logs a user out and blacklists the auth token. The token will be invalid. The user's refresh token is also regenerated.

    Request With Valid Token

    endpoint: POST '/auth/logout'

payload: { auth: { refresh_token: 'valid token' } }

header: { Authorization: 'Valid Auth Token' }

    Response

    status: 204

body: {}

    Request With Wrong Cridentials

    endpoint: POST '/auth/logout'

payload: { auth: { refresh_token: 'invalid token' } }

header: { Authorization: 'Valid Auth Token' }

    Response

    status: 422

body: { errors: 'Invalid Access Token' }

    Refresh Token

    The refresh endpoint takes the logged in user's refresh token as the request payload. This refreshes a user's auth token and extends its validity time.

    Request With Valid Token

    endpoint: POST '/auth/refresh'

payload: { auth: { refresh_token: 'valid token' } }

    Response

    status: 200

body: { jwt: 'valid auth token' }

    Request With Wrong Cridentials

    endpoint: POST '/auth/refresh'

payload: { auth: { refresh_token: 'invalid token' } }

    Response

    status: 422

body: { errors: 'Invalid Access Token' }

    Timer

    The timer endpoint takes seconds, start_at and stop_at as the request payload. This saves a record of the time logged. The jwt auth token MUST be included in the header.

    Request With Correct Data

    endpoint: POST '/timer'

payload: { timer: { seconds: 2613, start_at: '2018-09-28 19:10:16', stop_at: '2018-09-28 19:10:19' } }

header: { Authorization: 'Valid Auth Token' }

    Response

    status: 201

body: { timers: 
        [ 
            { id: 1, user_id: 1, seconds: 2613, start_at: '2018-09-28 19:10:16', stop_at: '2018-09-28 19:10:19', created_at: '2018-09-29 13:49:52', updated_at: '2018-09-29 13:49:52' },
            { id: 2, user_id: 1, seconds: 2613, start_at: '2018-09-28 19:10:16', stop_at: '2018-09-28 19:10:19', created_at: '2018-09-29 13:49:52', updated_at: '2018-09-29 13:49:52' },
            { id: 3, user_id: 1, seconds: 2613, start_at: '2018-09-28 19:10:16', stop_at: '2018-09-28 19:10:19', created_at: '2018-09-29 13:49:52', updated_at: '2018-09-29 13:49:52' },
            { id: 4, user_id: 1, seconds: 2613, start_at: '2018-09-28 19:10:16', stop_at: '2018-09-28 19:10:19', created_at: '2018-09-29 13:49:52', updated_at: '2018-09-29 13:49:52' }
        ] 
      }

    Request With Wrong Cridentials

    endpoint: POST '/timer'

payload: { timer: { seconds: 2613, start_at: '2018-09-28 19:10:16', stop_at: '2018-09-28 19:10:19' } }

header: { Authorization: 'Invalid Auth Token' }

    Response

    status: 422

body: { errors: 'Invalid Access Token' }

    Request With Wrong Data

    endpoint: POST '/timer'

payload: { timer: { seconds: nil, start_at: nil, stop_at: nil } }

header: { Authorization: 'Valid Auth Token' }

    Response

    status: 422

body: { errors: "Seconds can't be blank, Start at can't be blank, Stop at can't be blank" }
